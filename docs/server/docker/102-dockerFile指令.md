# 102-dockerFile指令


## 1、dockerFile的语法
dockerFile的常见几个语法:
* `FROM [镜像名称]:[版本号]`: 定义使用哪个版本的镜像启动镜像
* `MAINTAINER [用户信息]`: 定义镜像维护者的信息
* `LABEL [键] [值]`: 镜像描述元信息，可以写多条
* `ENV [键] [值]`: 设置环境变量，可以写多条
* `RUN [指令]`: 构建容器时需要运行的命令，可以写多条
* `WORKDIR [路径]`: 设置终端默认登录进来的工作目录 
* `EXPOSE [端口]`: 当前容器对外暴露出的端口
* `ADD [宿主机路径] [镜像内路径]`: 将宿主机的文件复制到容器内，如果是压缩文件，会自动解压
* `COPY [宿主机路径] [镜像内路径]`: 和ADD相似，但不会自动解压压缩文件
* `VOLUME`: 创建数据卷，由于编写dockerFile的时候，不能保证所有宿主机都有指定路径，所以在编写dockerFile的时候，就执行镜像里面的目录就可以了
* `CMD`: 指定容器启动时要运行的命令，假如有多个CMD，最后一个生效
* `ENTRYPOINT`: 指定容器启动时要运行的命令
* `ONBUILD`: 触发器，当构建一个被继承的Dockerfile时运行的命令，父镜像在呗子镜像继承后，父镜像的onbuild被触发


## 2、构建
编写完dockerFile后，执行`docker build -f [编写的dockerFile文件名称] -t [要生成的镜像名称]:[要生成的版本号] .`

> 注意: 最后的 `.` 不能省略
